# 大型营销服务（专栏）

## 项目定位

- 目标：用 DDD（领域驱动设计）落地营销活动/抽奖等业务
- 特点：规则复杂、策略多、并发高、数据一致性要求强

## 领域视角（建议拆分）

- 活动域：活动创建、审核、发布、生命周期
- 策略域：抽奖策略装配、权重与概率分发
- 规则域：前置/中置/后置规则过滤与组合
- 库存域：不超卖、可回收、可观测

## 关键实现记录（摘录）

### 策略概率装配

思路：

- 将奖品概率离散为“位置数组/列表”
- 通过乱序分布来实现 O(1) 抽取
- 结果与权重策略写入 Redis，运行期快速读取

### 规则过滤：责任链

适用场景：规则可插拔、顺序可编排、链式短路。

### 规则模型：决策树

适用场景：规则需要按节点条件分支，最终导向某个结果（奖品/兜底）。

### 库存不超卖（Redis）

建议：

- 先装配库存到缓存
- 扣减采用原子操作 + 凭证/过期自愈
- 异步落库与对账兜底
